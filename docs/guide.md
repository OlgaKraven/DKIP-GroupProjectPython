# Guide — Руководство по групповому проекту «Инвестиционный Аналитик ИС (InvestCalc)»

## 1. Цели проекта
Создание мини-ИС для оценки экономической эффективности ИС:
- TCO, ROI, Payback Period;
- сравнение локального и облачного сценариев;
- анализ чувствительности;
- генерация отчёта;
- запуск через Docker.

## 2. Команды и роли
Команда 2–4 человека. Возможные роли:
- аналитик;
- разработчик API;
- архитектор;
- DevOps;
- специалист по отчётам/презентации.

## 3. Этапы
1. Предметная область и модель (данные, таблицы, расчёты).
2. Архитектура и первый эндпоинт (`/calc/local`).
3. Анализ чувствительности, отчёты, Docker.
4. Финальная сборка и презентация.

## 4. Критерии оценивания (10 баллов)
- Бизнес-логика и расчёты — 3
- API и архитектура — 2
- Docker — 1
- GitHub (структура + README) — 1
- Рабочий прототип — 1
- Презентация — 2
